Protocol Description

Quick Crypt follows the protocols described below to encrypt, validate, and decrypt data. It is provided to help you understand how Quick Crypt works and allow you to make your own assessment of strength and trustwothiness. If you believe the protocol could be improved, please file a ticket.

Quick Crypt uses WebAuthn to create and authenticate user identities. Since this is a standard protocol described in detail elsewhere, include the mozilla developer network, webauthn is described only at a high level to show how user ids and user credentials are created and retrieved for use in cryptographic functions.
